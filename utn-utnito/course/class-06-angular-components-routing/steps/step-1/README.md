# Step 1 - Concepto de routing (proyecto aparte)

## Espanol

### Objetivo
Entender routing sin mezclar login/chat.

### Proyecto a usar
- `./routing-concept-demo`

### Que mostrar
- `ComponentOne` y `ComponentTwo`.
- Navegacion por UI con `routerLink`.
- Navegacion por logica con `router.navigate()`.
- Render de rutas con `router-outlet`.
- URL cambia sin recargar toda la pagina.

### Idea clave
- `routerLink`: el usuario navega haciendo click en la UI.
- `router.navigate()`: la aplicacion navega desde codigo.
- En aplicaciones reales se usa mucho `router.navigate()` (login, logout, redirects).

### Validacion
- `/one` muestra `ComponentOne`.
- `/two` muestra `ComponentTwo`.
